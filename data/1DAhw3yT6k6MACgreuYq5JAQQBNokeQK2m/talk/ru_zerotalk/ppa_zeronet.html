Концепт. Раздача установочных пакетов через ZeroNet

kostanew: (Mar 09, 2016)
Сейчас у пакетных дистрибутивов есть несколько бед с пакетами не из основного репозитория:

  1. 1. Для Ubuntu это выключение серверов раздающих редкие штуки по PPA. (Я так при переезде на неубитую систему потерял софт для синхронизации с Windows Phone 7, искал его потом несколько дней)
  2. Для CentOS, Scientific Linux, Fedora очень часто приходится искать редкий пакет на специальных сайтах, качать и ставить руками.
  3. Необходимость вводить аж три строчки в консоль, если надо подключить PPA или что то поставить каждому новому члену команды/научной группы. И главное они выглядят абсолютно не мнемонично, надо обязательно искать название или где то его записать.
  4. Отстойнейшее описание пакетов... Серьезно. Я должен пролазить 100500 сайтов что бы найти пакет для мелкой задачи. Например, я несколько часов искал программу Disk Usage Analyzer. Просто потому что я не знал ни существования концепции, ни названия, ни как объяснить google что же я хочу делать. Это просто жесть по сравнению со Steam или другими сервисами распространяющими контента, которые сами предлагаю штуки и удобный поиск по категориям.

Короче идея: сделать плагин, который выдает API для запуска .deb (и других) пакетов, прокаченых через 0net. Человек заходит на сайт, тыкает кнопку, пакет ставится. Проблема не решается в web, так как PPA сервер может прилечь навсегда, если распространитель софта на него забъет.
