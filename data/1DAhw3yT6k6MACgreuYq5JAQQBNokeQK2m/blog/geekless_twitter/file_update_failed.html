File update failed на несуществующих файлах

Интересная особенность. Если на зайте открыть URL несуществующей страницы (например), то ZeroNet сообщает для этого зайта, что:

File update failed

И в боковой панели соответственно:

Needs to be updated:
• foobar

Сообщения об этих фантомных файлах остаются, пока зайту не сделать Update. После этого ZeroNet понимает, что всё в порядке, весь контент на месте.

Сложно сказать, такое поведение было специально сделано с какой-то целью, или это несущественный побочный эффект где-то в коде.

Комментарии:

gitcenter: (May 06, 2018)
ZeroNet не знает, существует ли файл, так как он может быть опциональным, не скачаться, content.json может быть старым и т.д. Когда жмешь Update, он пытается этот файл+content.json скачать, видит, что ни у кого этого файла нет и успокаивается.

geekless: (May 06, 2018)
gitcenter: ... content.json может быть старым и т.д.
content.json может быть старым, но пока он не обновился, он и есть истина. Понятно, с одной стороны, этот File update failed показывает, что «может быть, после обновления файлы найдутся».

В другой, если где-то остались битые ссылки, пользователь всё время будет видеть этот индикатор сбоя — то есть с точки зрения пользователя «сбоит индикатор сбоя».
