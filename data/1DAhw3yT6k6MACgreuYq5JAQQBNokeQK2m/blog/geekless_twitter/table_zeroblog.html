Таблицы в ZeroBlog — бида-пичаль

Мало того, что таблицы в markdown — само по себе боль, так еще и встроенный в ZeroBlog редактор не поддерживает таблицы с переводами строк. Ни таблицы, сделанные при помощи конструкции |--|--|, ни таблицы в виде HTML-кода, не работают. Вернее, в первых редактор сразу портит разметку, а вторые сначала превращает в первые и затем тоже портит.

Если открыть режим ввода неформатированного текста, вставить таблицу и сразу сохранить пост, то таблица сохраняется нормально. Пробую:

X Y
first line
next line		 first line
next line
first line
next line		 first line
next line

Но если попытаться что-то еще отредактировать, всё разъезжается.

Вот markdown валидной таблицы:

| X | Y | | ----------------------- | ----------------------- | | first line
next line | first line
next line | | first line
next line | first line
next line |

А превращается он в это:

| X | Y | | --- | --- | | first line | next line | first line next line | | first line next line | first line next line |
