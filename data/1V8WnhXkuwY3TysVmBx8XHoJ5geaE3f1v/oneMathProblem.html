对“求N个随机数使其和固定”的实现方式的探究

0x00 前言

最近碰到一个这样的需求，数值表要求如下：

最小攻击力 最大攻击力 最小防御力 最大防御力 最小体力 最大体力 最终战力
400 480 200 240 900 1080 1600
战力=攻击力+防御+体力

其实就是战力固定，需要分解出攻击力、防御、体力

0x01 需求分解

  • 1、随机生成N个数，使得它们的和固定为S
  • 2、随机生成N个数，且\(n_i\in[a,b]\),同时使得它们的和固定为S
  • 3、随机生成N个数，且\(n_i\in[a_i,b_i]\),同时使得它们的和固定为S

0x02 第一种情况

方法一：剩余随机法

  • 1、随机生成第一个数\(n_1=random( 0,S )\)
  • 2、随机生成第二个数\(n_2=random(0,S-n_1)\)
  • 3、随机生成第三个数\(n_3=random(0,S-n_1-n_2)\)
  • 4、如此类推，生成最后一个\(n_N=random(0,S-\sum_{i=1}^{N-1}n_i) \)
function n_random(N,S)
    local rlist = {}
    for i=1,N-1 do
        local r = 0
        if S ~= 0 then
            r = math.random(S)
        end
        S = S - r 
        rlist[i] = r
    end
    rlist[N] = S
    return rlist
end

运行几次可以得到结论，后面几项非常大的概率为0，该方式不够随机

方法二：缩放法

  • 1、随机生成\(N\)个浮点数\(p_i=random(0,1)\)
  • 2、令\(n_i =\frac{p_i}{\sum_{j=1}^{N}p_j} * S \)
function n_random(N,S)
    local plist = {}
    local rlist = {}
    local psum = 0
    for i=1,N do
        plist[i] = math.random()
        psum = psum + plist[i]
        print(plist[i])
    end
    
    for i=1,N do
        rlist[i] = tonumber(string.format("%.0f",plist[i] * S / psum ))
    end
    return rlist
end

该方法可以近似均匀，不过好像还不是很随机

方法三：打点取段法

将一个长度为S的线段，随机取N-1个点，那么就将其分成了N分，那么这N个长度则是我们所需要的。

function n_random(N,S )
    local list = {}
    local rlist = {}
    for i=1,N-1 do
        list[i] = math.random(S)
    end
    table.sort(list)    --排序
    rlist[1] = list[1]
    rlist[N] = S - list[N-1]
    for i=2,N-1 do
        rlist[i] = list[i] - list[i-1]
    end
    return rlist
end

这种方法仿佛是最好的随机的方法

0x03 第二种情况

这种情况跟前一种的唯一区别是对单个随机数限定了范围\([a,b]\) 我们不妨转化一下问题为

随机生成N个数，且\(n_i\in[0,b-a]\),同时使得它们的和固定为\(S-a*N\)

即

随机生成N个数，且\(n_i\in[0,c]\),同时使得它们的和固定为\(s\)

方法一：剩余随机法

同上一种

math.random(S) 改为 math.random(c) 即可

方法二：随机扔球法

将问题看成是

有N个桶，每个桶的容量是c，将s个球随机扔在桶里，最后每个桶里面球的数量组成一组随机数

function n_random(N,S,c)
    local rlist = {}
    local plist = {}
    for i=1,N do
        --初始化
        plist[i] = i
        rlist[i] = 0
    end

    for i=1,S do
        local id  = math.random(#plist)
        local index = plist[id]
        rlist[index] = rlist[index] + 1
        if rlist[index] >= c then
            --已经满了
            table.remove(plist,id)
        end
    end

    return rlist
end

这种方式应该属于很好的一种随机方式，但是缺点就是效率问题

0x04 第三种情况

这种情况是覆盖前两种情况的，具有普遍意义

依然可以转化成上一种情况去处理

随机生成N个数，且\(n_i\in[0,b_i-a_i]\),同时使得它们的和固定为\(S-\sum_{i=1}^{N}a_i\)

PS：本文代码全部使用lua，不考虑没有解的情况，因为那种是可以提前判断出来的，比如\(N*a>S\)
