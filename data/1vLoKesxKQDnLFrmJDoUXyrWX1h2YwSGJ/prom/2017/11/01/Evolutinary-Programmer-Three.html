Evolutionary Programmer: October Summary

November 01, 2017

It’s been a while since I did a blog update on the Evolutionary Programmer. However much progress has been made.

What’s done

Noun phrase arguments

Since the last teaching added the capacity of having a number to plus for each ceremony. That means can do the toLower conversion as a one liner, for example plus 0x20.

Conditional Predication

I also added a conditional and adjusted the training set so that it wont toLower inputs that are below 0x40. It took some time to get the evolutionary programmer to get good at narrowing down the optimal value.

It is conditional predication because the condition comes at the begining of an independent clause and only affects the rest of the independent clause.

Synthetic Annealing

In fact I had to introduce synthetic annealing, which is a temperature value, similar to how the universe cools down over time. Except here the evolutionary programmer cools down with each generation, in practical terms that means the amount by which it mutates variables lowers with the drop in temperature. That way once the temperature drops to zero it only tries to adjust values by one, as hopefully by then they are pretty close anyways.

Multi-conditional predication

With toLower there is a lower bound but there is actually also an upper bound, as don’t want it to toLower numbers higher than Z.

So have support for using and connector. Now can do the whole toLower function, in one line.

What’s upcoming?

Since tuning these three settings is taking over 200 generations, it’s time to update the evolutionary algorithms again.

This time can use multiple islands in parallel via OpenCL. Will see if it can lower the generation count.

After that, also have to add the ability to invert functions, and then we’ll add for loops.

Stay tuned for next time.

Inverting Predication

Inverting predication should be fairly simple.

the toLower function

0x5B _num _abl down _con _and 0x40 _num _abl high _con 0x20 _num plus _rea

If we apply the contents of the predication onto the conditionals, and then invert the contents we get the toUpper function

0x7B _num _abl down _con _and 0x60 _num _abl high _con 0x20 _num subtract _rea
