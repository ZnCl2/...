Freenet Message System （FMS，Freenet 消息系统）是 Freenet 网络上的论坛系统。您可以点击下面的 ActiveLink 下载：

FMS 新手教程

本教程会教您 FMS 基本的用法。为了简单，我省略了不重要的内容。

假设

为了保持教程的简短，我假设：

  • 您已经成功下载、安装和启动了 FMS。
  • 您知道如何使用 Web 浏览器。
  • 当然，Freenet 也运行起来了。

我还假设 FMS 运行在本地主机，并监听端口 8080。如果您修改了此默认设置，您应该知道哪里需要调整。

本教程中到 FMS 本地界面的链接，因为与 Freenet 端口不同，会被认为是外部链接。点击后 Freenet 会显示警告消息。这是正常的。您只要确认一下链接的确链接到您的本地 Web 界面即可。或者，您可以将链接复制到浏览器地址栏直接访问。

此教程基于 FMS 0.3.65。它至少部分适用于未来的新版本。

1. Web 界面

当 FMS 运行时，您可以通过在浏览器访问 http://127.0.0.1:8080 以访问 FMS 的 Web 网页管理界面。为了以后访问方便，您可以将此网址加为浏览器书签。

2. 论坛界面

点击 Browse Forums （http://127.0.0.1:8080/forummain.htm）链接，您会进入论坛界面。

您的大部分时间会花在这里。如果您熟悉普通的论坛，您应该很快会使用其中的看帖和发帖的功能。

如果您是刚刚安装并启动 FMS，您会发现这里几乎是空的，什么内容都没有。别烦，这是正常的。和普通论坛不同，FMS 并不会连接到一个中心的储存内容的服务器。FMS 的帖子（还有用户、版面等）需要持续的、一部分一部分的从 Freenet 获取。这意味着如果您等数分钟或数小时，您就会看到很多帖子和用户了。全新安装的 FMS 需要下载所有存在的内容，这就需要一定的时间。我建议如果是全新安装的 FMS，您最好将它开一到两个小时，让它下载一些数据。

理想情况下，您可以一直开着 FMS，以获得最佳性能。如果您每次只开一会儿 FMS，别想立即看到所有帖子。

3. 添加版面

您可能奇怪为什么论坛里面没有几个版面。其实版面有很多，只是 FMS 自己没有添加它们。点击 FMS 界面上的 Board Maintenance 链接（http://127.0.0.1:8080/boards.htm）。

此页面列出了所有 FMS 知道的版面。请记住，这个列表随着 FMS 的运行，会不断增长。要将某个版面添加到论坛界面，请选中“Forum”列上的复选框，并点击“Update”按钮。

看，版面已经添加，可以看帖和发帖了。FMS 会自动发现版面，您不需要做什么事情。当然，您也可以自己创建版面。

4. 用户

FMS 持续从网络上了解其它用户的信息。Peer Maintenance（http://127.0.0.1:8080/peermaintenance.htm）页面显示 FMS 已经了解到的用户信息。

Recently Added Peers（http://127.0.0.1:8080/recentlyadded.htm）页面显示 FMS 最近发现的用户。随着 FMS 的运行，此列表会不断增长。

5. 帖子

如果 FMS 发现了某个用户，它会尝试抓取这个用户发的帖子。帖子不是一次性获取的。您可以在 Received Messages（http://127.0.0.1:8080/showreceivedmessage.htm）页面看到目前获得的帖子信息。

此页面显示了所有收到的帖子，包括您没有添加到论坛界面的版面中的帖子。

6. 创建一个用户

在您看了一段时间帖子后，您可能想自己发帖了。这时候您就需要创建一个用户。请进入 Create Identity（http://127.0.0.1:8080/createidentity.htm）页面。

输入您想要的名称，点击“Create”按钮，并稍等片刻。

您的新用户会显示在 Local Identities（http://127.0.0.1:8080/localidentities.htm）页面。

7. 通报您的用户信息

FMS 仅会抓取它知道的用户的帖子。如果您想其他人收到您发的帖子，您需要向他们通报您的用户的存在。不过您只需要通报一次即可。通报的方式是解谜，比如识别图像验证码。请到 [Announce Identity](http://127.0.0.1:8080/announceidentity.htm]（http://127.0.0.1:8080/announceidentity.htm）页面。

输入图像或语音验证码，并点击“Announce”按钮。您不需要识别和输入所有的验证码，也不需要保证每个验证码都填对（填错了没有惩罚）。最多识别 10 到 20 个验证码。

在页面底部 FMS 提供了一个选项，您可以为向您提供验证码的用户，添加对信任列表的信任。因为 FMS 是从其他用户的信任列表中获得新用户的，因此您最好选中此复选框，以快速发现其他用户。别担心，您以后随时可以回顾您的信任列表设置。（要了解更多有关“信任”的内容，请阅读信任问答。）

一段时间后（几小时，可能一天），Local Identities（http://127.0.0.1:8080/localidentities.htm）页面会显示您的用户状态为已通告（“Announced?”列显示“Yes”。这就可以确定其他人知道了您这个用户，并可以收到您发的帖子。（您可以在用户已通告之前就发帖子，但是这个时候您不能绝对肯定其他人能够看到您的帖子。）如果您的用户还没有被通告，请重复上面的步骤。

8. 发帖

进入论坛页面，使用您的新用户登录。进入“test”版面，点击“New Post”，撰写您的帖子，然后点击“Send”按钮。

搞定！您的帖子会被插入到等待发帖队列（http://127.0.0.1:8080/showpendingmessage.htm）。几分钟后，您的帖子会被插入到 Freenet。如果一切正常，过段时间，它会在“test”版面显示出来。这个时间可长可短，数分钟到数小时都有可能。请记住，Freenet 上发帖不是即时可见的。

9. 用户列表和节点信任

您可以在 Peer Trust（http://127.0.0.1:8080/peertrust.htm）页面管理信任级别。您也可以在此页面查看 FMS 已知的用户列表。大部分列是可以排序的。

信任是 FMS 系统设计中的重要部分。如果您准备好了解更多关于“信任”的信息，请阅读“信任问答”。

就这样吧。通过本教程希望您能学会 FMS 的基础。FMS 还有更高级的功能和选项。我会在以后的教程更新中不断添加。您也可以在其他网站或文章中了解到它们的用法。
