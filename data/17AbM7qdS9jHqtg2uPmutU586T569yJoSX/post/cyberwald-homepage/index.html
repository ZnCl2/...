Cyberwald Homepage

July 22, 2017 Linux, Homepage

Es mag nicht sofort auffallen aber unsere CyberWald Homepage unterscheidet sich stark von vielen anderen Seiten im Internet.

Die CyberWald Homepage wird über das “Inter Planetary File System” (IPFS) gehostet.
Das bedeutet, dass unsere Homepage nicht auf einem zentralen Server liegt, sondern über ein Peer-to-Peer-Netzwerk verteilt wird. Bei einem P-2-P-Netzwerk tauschen die teilnehmenden Rechner die Daten direkt aus und brauchen dafür keinen zentralen Server. Ruft man unsere Homepage über das IPFS-Netzwerk auf dann kann die Webseite von überall auf der Welt übertragen werden. Wenn man z.B. über einen öffentlichen Hotspot verbunden ist und ein Rechner dort bietet unsere Homepage an, dann wird nicht mal eine Internet Verbindung benötigt um unsere Homepage zu öffnen.

Im Gegensatz zum klassischen Client-Server-Modell steht nicht der zentrale Server mit seinen Diensten im Vordergrund sondern die bereitgestellten Daten.
Bei IPFS spricht man auch nicht von Daten oder Dateien sondern von Objekten und diese Objekte haben einen Hash, also so etwas wie ein einmaliger Fingerabdruck.
Möchte man nun eine bestimmte Datei (Objekt) öffnen, dann fragt man im Netzwerk nicht nach einen Server (www.cyberwald.com) + Dienst (Webserver) + Datei (textdatei.txt)
http://www.cyberwald.com/textdatei.txt
Sondern man fragt einfach nach dem Hashwert:
QmWKt38vVcwy9W8s9oWKonoJ3bccPWZfv8h2ggabNcahns (textdatei.txt)

Warum setzten wir nun auf IPFS?:

Ein zentraler Dienst / Server kann:

  • durch einen defekt ausfallen.
  • aufgrund von DDoS angriffen nicht oder kaum erreichbar sein.
  • manipuliert und zensiert werden.

IPFS hat all diese Nachteile nicht und daher liegen die Vorteile klar auf der Hand.

Damit man unsere CyberWald Homepage über jeden Browser aufrufen kann nutzen wir einen Gateway-Server. Dieser Server fragt die Daten im IPFS-Netzwerk an und gibt diese dann als „normale“ Homepage aus.

Wer uns direkt über IPFS erreichen möchte muss dafür den IPFS-Client installieren und kann uns dort über: /ipns/cyberwald.com erreichen.
Wer uns helfen möchte, kann unsere Homepage „pinnen“ und so mit zur Verbreitung im IPFS-Netzwerk beitragen:

ipfs pin add /ipns/cyberwald.com

ipfs pin add /ipns/www.cyberwald.com

Eine weitere Besonderheit unserer Homepage entsteht durch den Verzicht auf dynamische Webseiten.
Bei uns kommen keine verwundbaren Server Technologien wie Java, PHP oder Python zum Einsatz sondern nur statische Webseiten. Selbst wenn wir einen zentralen Server hätten, dann wäre dieser durch Sicherheitslücken in den genannten Technologien nicht verwundbar.
Wir verwenden das Hugo-Framework und generieren damit unsere CyberWald Homepage.

Unsere Homepage ist im Vergleich zu vielen anderen Webseiten schon sehr außergewöhnlich aber da geht noch mehr. In Zukunft werden wir auch im [ZeroNet][https://zeronet.io] vertreten sein.

Neue Technologien wie Hugo, IPFS und ZeroNet werden wir in Zukunft noch genauer betrachten.
Daher bleibt uns treu und habt viel Spaß auf der CyberWald Homepage.
