Quick Command-Line View Of i2pSnark Torrents

by Miracle Max

Posted on June 19, 2018

Ever want to check the status of your i2pSnark torrents without firing up a browser? I did, and hereâ€™s a python script I made to do that.

snark

#!/usr/bin/python3
import requests, sys, re
from bs4 import BeautifulSoup
from pprint import pprint

mode = 'NotSeeding' #default
searchstring = None

if len(sys.argv) > 1:
    if sys.argv[1] == '-h' or sys.argv[1] == '--help':
        print("Usage 1: snark [option]")
        print("Usage 2: snark command <command argument>")
        print("Options:\n\t<none>\tShow torrents that are not seeding (defualt)\n\t-a\tAll\n\t-s\tSeeding")
        print("Commands:\n\tfind <searchstring>\tShow torrents with searchstring in name.")
    if sys.argv[1] == '-s':
        mode = 'Seeding'
    elif sys.argv[1] == '-a':
        mode = 'All'
    elif sys.argv[1] == 'find':
        mode = 'All'
        searchstring = sys.argv[2]
    else:
        mode = 'NotSeeding'

data = []

def getPage(st):
    global data
    r = requests.get('http://127.0.0.1:7657/i2psnark/?sort=2&st=%s' % st)
    #pprint(r.text)
    soup = BeautifulSoup(r.text, 'html.parser')
    #pprint(soup)
    div = soup.find('div', attrs={'id':'mainsection'})
    table = div.find('table', attrs={'class':'snarkTorrents'})
    
    tableBody = table.find('tbody')
    if tableBody:
        rows = tableBody.find_all('tr')
    else:
        rows = table.find_all('tr')
    for row in rows[1:]:
        if not row.find('td'):
            #all headers
            continue
        torrentStatus = row.find('td', attrs={'class':'snarkTorrentStatus'}).text.split(':')[0]
        try:
            torrentName = row.find('td', attrs={'class':'snarkTorrentName'}).contents[0].text
        except:
            torrentName = row.find('td', attrs={'class':'snarkTorrentName'}).contents[0]
        #print(torrentName)
        torrentETA = row.find('td', attrs={'class':'snarkTorrentETA'}).text
        torrentRate = row.find('td', attrs={'class':'snarkTorrentRateDown'}).text
        data.append([torrentStatus,torrentName,torrentETA,torrentRate])
    return len(rows)

#loop through pages
pageSize = 50
rowcount = pageSize + 1
st = 0
while rowcount > pageSize:
    rowcount = getPage(st)
    print("rowcount: %s" % rowcount)
    st += pageSize


#pprint(data)

for d in data:
    if len(d) > 1:
        torrentStatus = d[0]
        torrentName = d[1]
        if searchstring and searchstring not in torrentName:
            continue
        if torrentStatus == 'OK':
            #pprint(d)
            timeRemaining = d[2]
            downloadSpeed = d[3]
        else:
            timeRemaining = 0
            downloadSpeed = 0
        if mode == 'All' or torrentStatus == mode:
            if timeRemaining:
                print("%s: %s (ETA: %s @ %s)" % (torrentStatus,torrentName,timeRemaining,downloadSpeed))
            else:
                print("%s: %s" % (torrentStatus,torrentName))
        elif mode == 'NotSeeding' and torrentStatus != 'Seeding':
            if timeRemaining:
                print("%s: %s (ETA: %s @ %s)" % (torrentStatus,torrentName,timeRemaining,downloadSpeed))
            else:
                print("%s: %s" % (torrentStatus,torrentName))

Background

So it seems there used to be more command-line to i2psnark, but as far as I can tell, in the current i2p implementation, there isnâ€™t much useful; everything is done through the web interface. So thatâ€™s what we have to work with.

The script scrapes the web interface and lets you list just the non-seeding torrents (defualt), seeding torrents (snark -s), all torrents (snark -a), or torrents that contain a search term (snark find term). The script will attempt to scroll through multiple pages of torrents adjust the pageSize variable if you use something other than the default of 50.

Curious note: Using a page-size of > 50 caused midori to crash (along with the x server) on my setup.
