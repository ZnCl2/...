Using The Midori NoJS Extension With I2P

by Miracle Max

Posted on May 18, 2018

Uncheck Set permissions on second-level domain in the NoJS preferences or else it won’t recognize i2p sites.

Background

I use midori as my main browser. It’s small, quick, and loads fast. It also comes with some useful extensions such as an ad-blocker and an extension that controls whether javascript is allowed on a site-by-site basis (NoJS).

By default, NoJS will filter by second-level domain. So enabling javascript on widgets.com will enable javascript on scripts.widgets.com, foo.widgets.com, etc.

I had had some reservation about using midori along with i2p and tor because of this, but after some experimenting, I found this work-around.

I think the root issue is that whatever library midori uses to report the top-level domain, just doesn’t work with .i2p addresses.
