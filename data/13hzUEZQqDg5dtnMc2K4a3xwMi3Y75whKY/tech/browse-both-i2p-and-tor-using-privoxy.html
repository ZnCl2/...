Browse Both I2P And Tor Using Privoxy

by Miracle Max

Posted on January 27, 2018

You can browse I2P and Tor with the same browser if you use Privoxy to choose which one to forward requests to.

Privoxy is a light-weight filtering proxy that can be configured to forward requests to other proxies based on the url and/or port.

The config file actually comes with instructions for setting it up for Tor. Here’s how you can set it up to use I2P for .i2p addresses, use no proxy for local address, and pass everything else to Tor:

forward-socks5t / 127.0.0.1:9050
forward .i2p/ 127.0.0.1:4444
forward .i2p:443/ 127.0.0.1:4445
forward 192.168.*.*/ .
forward 10.*.*.*/ .
forward 127.*.*.*/ .

The first pattern ‘/’ matches everything and sends it to Tor (port 9050), but Privoxy will use the last match. So any .i2p addresses will go to either the standard I2P port, or the SSL port instead. Lastly, if the IP matches any of the private networks, no proxy will be used (. = no proxy).

Save the config file, restart Privoxy, and set your browser to use 127.0.0.1:8118 as the proxy for all requests.

Tor Browser

This will mess up some of Tor-Browser’s plugins. It will tell you that you aren’t connected to Tor (even though you are) and you won’t be able to use the button to use a new tor circuit or identity.

I should note, that I’ve assumed that you’re using the stand-alone tor service, which uses port 9050. Tor browser comes with it’s own tor router, which uses port 9150. You could point Privoxy to that port, but it’ll only run when Tor Browser is running.
