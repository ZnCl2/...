Zeronet Build Out With Hugo

@punkcoder · Sep 19, 2017 · 2 min read

After going through and working with zeronet for a while, I thought I would share the process for getting a site up and running without the standard, clone site. Hugo, similar to Jekyll, is a fairly good option for doing this kinds of site since they create static sites.

I originally went out looking for solutions to create new sites I kept running into the “Get Started Quickly” process provided by the clone site button. While I like this option it doesn’t really satisfy the needs that I had in mind, there’s nothing compared to getting down into the guts of the thing and trying to figure out how all of the moving parts work.

So here is a list of the steps that I went through in trying to figure out how to do this.

  1. Create Site
  2. Create HUGO site
  3. Pull theme
  4. Build “build” script
  5. Remove external references
  6. Copy over zeroframe work
  7. Fill in content
  8. Sign site
  9. Deploy site ??? - I’m still not sure that I did this correct.

Problems that I ran into

  • All.js and all.css

I ran into several issues with the creation of all.js and all.css. I think I was able to fix this by restarting zeronet.

  • SitePublish

Not sure that I did this one right as the zeronet sign seemed to work correctly, but when I tried the sitePublish, I got no response. I’m wondering if this is because i am running the windows version. Update: Apparently you can update all of this now through the UI, grabbing the zeronet hover in the upper right hand now allows for the updating and signing of data directly on that site.
