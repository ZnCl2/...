Offline Zeronet

@punkcoder · Sep 21, 2017 · 2 min read

So I ran a bit of an experiment, I was running zeronet is an offline mode and I noticed that there are some sites that are severly broken in a full offline state. For me it’s a matter of taking the train, but it’s important to note the opperation of a network of communications in the attempt of a full government shutdown, such as the one that was seen in Turkey in 2016. For some decentralization is about the protection of servers, but I think the real goal at the heart of this is the protection of the flow of knowledge in the event of disruption.

In order to be truely decentralized the system needs to be able to support an offline mode. Additionally I haven’t tried it but it would also be interesting to note the ability for zeronet to operate in a closed LAN mode. I’m thinking that this will be my next experiment, and may result in me digging further into the code base to see if this is supported and if it isn’t what would need to be included to support this type of communication.

There are also a lot of zites that make references to off-zite content. While this is helpful in an alway online manner, it creates genuine issue for offline data.
